# 楽マッチ AI LP コンプライアンス チェックリスト

不動産 × AI SaaS のランディングページで踏みやすい法的地雷を、コピー作成・実装時に都度確認するためのチェックリスト。**根拠**: 景品表示法（景表法）／宅地建物取引業法（宅建業法）・不動産の表示に関する公正競争規約／特定商取引法（特商法）／個人情報保護法。プロダクト事実の裏取りは `../marketing-research/02_product_features_usp.md` の「言ってよいライン／危ないライン」を一次資料とする。

各表現を書くとき・レビューするときは、下の各セクションを上から確認すること。**OK例／NG例は実際に使える / 使ってはいけない文言**として書いている。

---

## 0. 使い方（コピー担当・実装担当・レビュー担当）

- コピーを1文書くごとに「これは事実か／効果の断定か／比較の最上級か」を自問する。
- 数値・割合・「No.1」「唯一」系・「保証」系・「無料」系は**全件、根拠と注記の有無をこのチェックリストで確認**してからコミットする。
- 注記（※印）は**本文と同じ画面・同じ視線の流れの中**に、本文と著しく差のない文字サイズで置く（打消し表示の要件）。フッターの極小文字に隔離しない。
- 迷ったら断定を避け、相対表現・プロセス表現に逃がす。**「盛らない」がLP全体の基本方針**。

---

## 1. 景品表示法（景表法）

### 1-1. 優良誤認（品質・効果の偽り／誇張）

実際よりも著しく優良だと誤認させる表示は禁止（景表法5条1号）。**成果・効果の断定と保証が最大の地雷**。

| 区分 | 例 |
|------|----|
| NG | 「成約率が上がる」「売上が伸びる」「反響が2倍になる」「絶対に追客漏れがなくなる」「AIが必ず最適な物件を提案」 |
| OK | 「追客の抜け漏れを減らす」「反響対応の手間を軽くする」「お客様の希望条件に合う物件を一覧で確認できる」「提案メールの下書きをAIが作成」 |

- **成果保証はしない**。「上がる／増える／伸びる」を主語が顧客の成果になる形で書かない。機能が「何をするか（プロセス）」に言い換える。
- 「必ず」「確実に」「絶対」「100%」など断定の副詞を効果に付けない。
- AIの精度・正確性を「完璧」「ミスしない」と書かない。AIは下書き・候補提示であり、最終判断は人間という前提を崩さない。

### 1-2. 有利誤認（価格・取引条件の偽り／誇張）

取引条件が著しく有利だと誤認させる表示は禁止（景表法5条2号）。**価格・無料・割引の条件不備が地雷**。

| 区分 | 例 |
|------|----|
| NG | 「初月無料！」（条件を併記せず） / 「月3,000円で使い放題」（利用枠・容量上限があるのに） / 「今だけ」（恒常的に出している場合） |
| OK | 「初月無料（招待限定クーポン適用時）」 / 「月3,000円/人（Standard・利用枠あり）」 / 価格は税込・人数課金である旨を明記 |

- **「無料」は条件を必ず近接表示**。楽マッチの初月無料は「招待限定クーポン」前提なので、無条件の無料と読める書き方をしない。
- 価格は**1人あたり・月額・税込/税別の別**を明記。プラン名（Standard / Premium）と対応づける。
- 「使い放題」「無制限」は利用枠・容量上限がある以上NG。
- 二重価格・割引を出すなら、比較対象の価格が実際の販売価格であること（架空の「通常価格」を作らない）。

### 1-3. No.1・唯一・最上級表示

「No.1」「業界唯一」「世界初」「最高」等は、**客観的な調査・出典がなければ全てNG**（景表法・No.1表示に関する考え方）。

| 区分 | 例 |
|------|----|
| NG | 「唯一のお客様連動アプリ」「業界初の逆引きマッチング」「No.1の追客効率」「他にはない機能」「世界初」 |
| OK | 「数少ないお客様連動アプリ」「お客様起点の逆引きマッチング」「競合では限定的な〜を標準搭載」 |

- 「唯一」「初」「最」「No.1」「一番」「日本一」は、**第三者調査の出典（調査主体・時期・範囲）を併記できないなら使わない**。
- お客様連動アプリは強い差別化だが、「唯一」と断定する裏付け資料は現時点でない（Facilo の対応状況は確認待ち）。**相対表現（「数少ない」「他社では限定的」）に留める**。

### 1-4. 打消し表示・注記の視認性

注記（※、注など）で本文の印象を打ち消す場合、**注記が見えなければ打消しとして無効**とされ、本文だけで誤認を招いたと判断される。

- 数値・効果に付ける「※自社利用に基づく試算」等は、**本文の直後または同一視界**に置く。
- 文字サイズが本文に対して極端に小さい／背景と低コントラスト／スクロールしないと見えない位置は**NG**。
- 1つの注記で複数の表現をまとめて打ち消そうとしない（どの表現にかかる注記か対応を明確に）。

## 2. 数値・統計・実績表現

LP内のあらゆる数値（精度%、効率化%、削減時間、件数）は、根拠と注記を必須とする。

| 区分 | 例 |
|------|----|
| NG | 「業務効率90%改善」「マッチング精度95%」「対応時間を半分に」（出典・条件なし） |
| OK | 「対応時間の削減を実感（※自社利用に基づく試算）」「マッチング精度 約95%（※自社の評価データに基づく社内測定値）」 |

- 数値を出すなら**注記で「誰が・いつ・どの条件で」測ったか**を示す。汎用に「※自社利用に基づく試算」を最低ラインとする。
- 第三者の成果を「お客様の声」として載せる場合、**実在・許諾・誇張なし**を確認。創作した体験談は景表法・ステマ規制違反。
- 「導入○社」等の実績数は、実数の裏取りがないなら載せない。

## 3. 宅建業法・不動産の表示に関する公正競争規約（一般原則）

楽マッチは仲介業者向けSaaSであり、LPが「物件広告」そのものではない。ただし不動産業界向けのサービス紹介として、**誇大広告・誤認を招く表示を避ける一般原則**は守る。

- LP上に具体的な物件・価格・利回りを「実例」として出す場合は、それが**デモ／サンプルである旨を明記**（実在物件と誤認させない）。
- 投資用物件マッチングに触れる際、**利回り・収益を保証するかのような表現はしない**（「高利回り物件が見つかる」等はNG。「利回り条件でのマッチングに対応」はOK）。
- サービスが宅建業務そのものを代替・保証するかのような表現（「重要事項説明をAIが代行」等、実態と異なるもの）はしない。

## 4. 楽マッチ固有の禁止表現 → 言い換え（プロダクト事実ベース）

`02_product_features_usp.md` で確定済みの「言い換え必須」リスト。**この6点はLP全ページで機械的にチェックする**。

| # | NG（禁止） | OK（言い換え） | 理由 |
|---|-----------|---------------|------|
| 1 | 「成約率が上がる」「売上が伸びる」 | 「追客の抜け漏れを減らす」等プロセス表現 | 成果保証は景表法・優良誤認 |
| 2 | 「唯一」「業界初」「世界初」 | 「数少ない」「お客様起点の」等の相対表現 | 根拠資料なしの最上級はNG |
| 3 | 「一切外部にアクセスしない」「データは外に出ない」 | 経路を分けて事実を説明（例: 「○○の処理は△△内で完結」） | 経路により異なるため一律断定は不正確 |
| 4 | 「売買・賃貸を1つのリストで同時管理」 | 「業態モードを切り替えて管理」 | 実態は業態モード切替であり同時一覧ではない |
| 5 | Premium「賢いAI」「高性能AI」「より精度の高いAI」 | 「利用枠と容量が拡大」 | プランによるAIモデルの差はない（差は枠と容量のみ） |
| 6 | 精度・効率化等の数値を裸で記載 | 数値に「※自社利用に基づく試算」等の注記を付す | 出典なき数値は優良誤認の恐れ |

### 補足: 各機能を「事実どおり」に書くための言い換え早見表

| 機能 | 言ってよい（事実） |
|------|--------------------|
| お客様連動アプリ | 「お客様自身がスマホで希望条件を更新でき、営業担当に共有される」 |
| 逆引きマッチング | 「お客様の希望条件を起点に物件を探せる」 |
| 通話録音 | 「電話商談を録音して文字起こしできる」 |
| 反響メール分類 | 「問い合わせメールをAIが種別判定する」 |
| 業態切替 | 「売買と賃貸で業態モードを切り替えられる」 |

## 5. 特定商取引法（特商法）・申込み導線

LPが申込み・課金導線を含む場合（CTA「無料で始める」「お申し込み」等）に確認する。

- **特商法に基づく表記**へのリンクが、申込み導線から到達可能な位置にあること（事業者名・所在地・連絡先・販売価格・支払方法・解約条件等）。
- CTAボタンの文言と、その先で実際に発生する課金・契約内容が一致していること（「無料で始める」の先が即課金ならNG）。
- 解約・返金条件を、申込み前に確認できること。
- サブスク（定期課金）である旨、課金タイミング・更新条件を明示（特商法の定期購入規制）。

---

## 6. 個人情報・お客様データの取り扱い表現

お客様連動アプリは**エンドのお客様（仲介業者の顧客）の希望条件等の個人データ**を扱う。ここの表現は特に慎重に。

- 「お客様のデータは一切外部に出ない／一切アクセスしない」と**断定しない**（→ セクション4 #3）。経路ごとに事実を分けて説明する。
- セキュリティ・プライバシーについて「完全に安全」「絶対に漏れない」と断定しない。「○○に配慮した設計」等、事実の範囲で書く。
- お客様データの利用目的・共有範囲（お客様→担当者への共有など）を、誤解なく説明する。第三者提供・AI学習利用の有無について事実と異なる印象を与えない。
- プライバシーポリシーへの導線を確保する。

---

## 7. その他・横断チェック

- **ステマ規制**: 「お客様の声」「レビュー」は実在・許諾必須。広告である旨が分かること。
- **誤認を招く比較**: 競合名（Facilo 等）を出す比較は、**事実かつ出典のある範囲**に限る。相手の機能を実態より低く書かない（Facilo の賃貸対応状況は確認待ちのため、断定的比較は保留）。
- **画像・図解**: スクリーンショットやデモ画面に**架空の好成績・実在しない数値**を写し込まない。旧ブランド表記「TOP-SALES AI」が残っていないか（現行は「楽マッチ AI」）。
- **将来機能を現在形で書かない**: Slack連携等の未実装機能を「できます」と書かない（「予定」と明記するか載せない）。

## 8. レビュー時のチェック項目（Phase 2 QA で使用）

page.tsx と /features/* 全ページに対して、以下を Grep/Read で機械的に確認する。

- [ ] 効果保証語（上がる／伸びる／増える／必ず／確実／絶対／100%／保証）の有無
- [ ] 最上級語（唯一／初／No.1／最高／一番／世界初／業界初／他にない）の有無
- [ ] 「一切」「完全」「絶対」+ セキュリティ/外部アクセス文脈の有無
- [ ] 「同時管理」「1つのリスト」等、業態切替の実態と異なる表現の有無
- [ ] Premium 説明に「賢い／高性能／高精度なAI」等モデル差を示唆する語の有無
- [ ] 裸の数値（%・倍・時間・件数）に注記が近接しているか
- [ ] 「無料」「初月無料」に条件（招待限定等）が併記されているか
- [ ] 価格に人数課金・月額・税の別が明記されているか
- [ ] デモ/サンプル画面である旨の明記があるか（実物件・実数値と誤認しないか）
- [ ] 特商法表記・プライバシーポリシーへの導線があるか
- [ ] 旧ブランド「TOP-SALES AI」表記が残っていないか
- [ ] 未実装機能を現在形（できます）で書いていないか
