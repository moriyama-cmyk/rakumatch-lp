# 楽マッチ AI トップLP刷新 完了ノート

更新日: 2026-07-20  
対象: `src/app/page.tsx` と `src/components/lp/**` を中心とするトップLP

## 1. 今回の結論

LPの訴求を「月額3,000円/人（税込・スタンダード）から × 5つの主機能 × 1名から」に統一した。最大の差別化は「お客様が保存した反応が担当者に届き、次の提案を考えられる」という流れとして、機能一覧より前に提示した。

匿名デモと7日間トライアルは別の開始方法として扱い、URL・文言・条件・計測イベントを分離した。

- 匿名デモ: 登録なし、ログイン・連絡先・カード不要
- 7日間トライアル: カード登録あり、7日以内の解約は料金なし、8日目から課金

## 2. 新しいセクション構成

1. **Hero** — 対象者、月額、1名から、5機能、2つの開始方法、実製品画面を最初に提示
2. **Problem** — 顧客・物件・お客様の反応が提案につながりにくい状態を3点に整理
3. **SignalFlow** — 「保存 → 担当者に届く → 先に提案」の因果を縦横に組み替わる3ステップで説明
4. **CapabilitySuite** — 5つの主機能、実画面、物件一括取り込みから候補化までの一本フロー
5. **Story** — 森山幸弘さんの実名・写真・現役不動産営業としての一次性
6. **Pricing** — 1名からの2プラン、共通5機能、匿名デモ、トライアル、初期支援
7. **Why** — 自社の公開条件と比較対象5社の「料金表示・開始方法」だけを事実列挙
8. **Voices** — 実在8件の本文を変えず、代表2件を初期表示
9. **Security** — 決済情報、削除方針、AI判断の位置づけなど、確認可能な事実
10. **FAQ** — 購入前の主要6問だけに圧縮
11. **FinalCta** — 匿名デモとカード付きトライアルの二択を再提示

Header、モバイル固定CTA、Footerは維持し、法定ページへの導線を残した。

## 3. 廃止・統合した構成

- **TrustStrip**: Heroの価格・1名・5機能・条件と重複するため、Heroへ吸収
- **EraShift**: 将来宣言や広い市場一般化を避け、1名から始められる事実へ置換
- **SolutionCore**: 概念説明の重複を避け、SignalFlowとCapabilitySuiteへ統合
- **旧CustomerApp / Matching / Ingest / AiPartner / FeatureDigest**: トップでは個別の長い章にせず、SignalFlowとCapabilitySuiteへ集約
- **旧Whyの横長8行比較表**: スマホ負荷と評価的表現を減らし、料金表示・開始方法だけの2列比較へ置換
- **補助3機能帯**: 5機能の主張を散らすためCapabilitySuiteから削除。詳細ページへの法定・既存導線はFooterで維持

旧コンポーネントのファイル自体は、詳細ページや履歴を壊さないため削除していない。トップページからの読み込みだけを外している。

## 4. 森山さん確認事項: 初期支援

LPには次の条件で記載した。

- 対象: 本ページ公開後の新規契約、先着5アカウント
- 時間・回数: 30分、1回
- 方法: オンライン
- 対応者: 開発者本人
- 範囲: 初期設定、最初の顧客・物件登録、お客様アプリの初回共有
- 連絡方法: 契約後、登録メールアドレスへ日程調整を案内
- 対象外: 入力代行、個別の営業判断、法務相談

この提供条件で実際に運用できるか、公開前に森山さんの最終確認が必要。

## 5. Voices初期表示を4件から2件へ変更した理由

8件の本文は履歴と照合し、1文字も変更していない。

2026-07-19の「3枚だと2列グリッドの右下が1マス空く」という指摘は正しい。今回は3枚ではなく2枚を初期表示にした。スマホは1列で2行、640px以上は2列で1行となるため、空きマスは発生しない。展開後も8枚なので2列で4行となり、空きマスは発生しない。

2件にした理由は、長文の声を料金直後に8件並べず、営業側1件・お客様側1件を代表表示して読了負荷を下げるため。残り6件は44px以上の「もっと見る」ボタンで展開できる。

## 6. 計測の変更

- `copyVariant`: 実表示と一致する `v4-density-signal-proof`
- 匿名デモ: `demo_cta_click`
- 7日間トライアル: `trial_cta_click`
- ページ内アンカー: `feature_anchor_click`
- Hero / お客様アプリ / 料金の到達: `section_view`
- 計測対象の全CTA・アンカー: `cta_location`、`copy_variant`、`link_url` を送信

LP側のコード整合までは確認した。GA4・Clarityでの実受信、アプリ側の `/try` 到達、`guest_start`、`signup`、`checkout_start`、課金継続への接続は、実環境で別途確認が必要。

## 7. ビルド・静的検査

- `npm run build`: 成功。トップ、法定ページ、詳細機能ページ、ブログを含む18ルートを生成
- `npm run lint`: 警告・エラーともに0
- `/privacy`、`/terms`、`/tokusho`: ビルド成果物に導線が存在
- Voices本文: `HEAD~2` と現行の8件を照合し、差分なし
- 最小文字: トップで使用する `text-xs` を14pxに定義。実表示トップに14px未満の任意サイズ指定なし
- タップ領域: CTAは44px以上、FAQは56px、Voices展開ボタンは44px。主要リンクも44px以上になるクラスを使用
- 横幅対策: `min-w-0`、折返し、レスポンシブ列変更、価格行の`flex-wrap`、セクション単位のはみ出し対策を実装

### 実描画検証の未完了

Chromeで `http://127.0.0.1:3000` を開いて375px / 768px / 1440pxを確認しようとしたが、ユーザー側のブラウザポリシーによりローカルURLが明示的に拒否された。別ブラウザ・別URLへの迂回も禁止されたため、回避していない。

そのため、横スクロールなし・14px以上・44px以上についてはコードとビルド成果物の静的確認までであり、実描画の合格とは報告しない。実ブラウザが許可された環境で、3幅のスクリーンショット、固定CTA、画像内文字、フォーカス順、コントラストを再確認する必要がある。
