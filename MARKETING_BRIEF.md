## 1. ポジショニング確定

### 1-1. ステートメント
> 中小・個人の不動産"売買"仲介にとって、楽マッチ AI は「物件を貼るだけで、入社初日の新人でもトップ営業の"動き"を再現できる」AI営業CRMである。高機能・高価格な大手CRM（初期数万〜20万・月5〜6万）と違い、必要十分な機能を低単価・初期0円で提供し、さらに競合にない「お客様連動アプリ（反応の逆流）」と「逆引きマッチング」で、追客のタイミングと相手を逃さない。

### 1-2. ターゲット階層（コピーのトーン基準・00 §1 と整合）
- **本命: 中小売買仲介の経営者・店長（2〜20名）**
  痛み＝トップ営業への依存（辞めたら売上が消える）／新人が一人前まで時間がかかる／追客・マッチングが属人化して漏れる。決裁権あり・ROIを語れる。
  響く言葉＝「新人がいきなり戦力に」「属人化を仕組みに変える」。
- **第二: 個人事業の売買エージェント／一人社長**
  痛み＝一人で物件入力・追客・顧客管理。雑務で商談時間が溶ける。最も摩擦が低くセルフサーブで獲れる。
  響く言葉＝「一人でも会社みたいに回る」「雑務はAIに、あなたは商談に」。
- **成長: 売買＋賃貸の兼業地域店**（主役にしない第二の入口）
  「賃貸兼業でも1本で両方」。売買で勝ってから横展開。
- **配置原則**: 見出し＝経営者視点（経営インパクト）、本文・ベネフィット＝現場視点（手間が減る）。上下で両者を取りに行く。主軸は経営者に寄せ個人はセカンダリ。
- **利用形態の二面を主要メリットに格上げ（森山さん追加方針 2026-05-30）**: 「個人でも企業でも」を“ついで”の併記にせず、**「自分の顧客を個人で管理したい時も、会社全体を管理したい時も、1本で両立できる」**を独立した主要ベネフィットとして打ち出す（専用セクション §8.5）。製品事実: ①1人でも企業でも同額 ¥3,000/人 ②会社利用時は権限分離=オーナーは全顧客を俯瞰、各担当者は自分の担当のみ（RLSで物理分離）。これは「個人プレイヤーの相棒」と「会社の標準ツール」の両方の入口を1つのLPで取りに行く打ち手で、ターゲット第二層(個人エージェント)を“格下げ”せず拾える。法務: 「成果保証」等のNGは維持（§8）。

### 1-3. なぜこのポジショニングか（01/02 の裏取り反映）
- 大手CRM（KASIKA / いえらぶ / PropoCloud 等）は高機能・高価格で中小売買には過剰かつ高い。楽マッチは「貼るだけ・低単価・初期0円」で導入障壁を壊す空白地帯を取る。
- 「お客様→担当者の逆方向動線」は競合がほぼ持たない（01の空白地帯）。ただし「唯一・だけ」は全数調査の裏付けがない限り断定せず相対表現に（§8）。
- 売買は成約単価が大きく属人化が激しい。「新人がトップ営業の動きを再現」は経営者の採用・育成コストの直撃ポイント。
- 注意（00 §10）: 「API契約不要」は単独の特長にしない（レインズに公開APIが無く業界共通の前提／PDF取込・メール生成・初期0は競合も持つ）。訴求は「契約も連携設定もいらず低単価で今日から（導入の軽さ×低単価）」に翻訳。

## 3. LPセクション設計（確定順序・役割・到達目標）

> 凡例: 【層】=§2のどの層／【目標】=読者に起こす状態変化／【現行差分】=現 page.tsx からの変更。デザインの具体（クラス・トークン・余白）は DESIGN_SYSTEM.md に従う。

### 確定順序（結論）
`Header → Hero → 統計バー → 社会的証明 → 課題 → 堀=お客様連動アプリ → 主要機能 → 通話録音 → 比較表 → 信頼補強(信頼/セキュリティ＋AI進化) → 利用形態(個人での管理／会社での全体管理＋担当者分離) → 料金 → FAQ → CTA → Footer/注記`

> 注: 「利用形態」セクションは信頼補強の後・料金の直前に新設（§8.5。森山さん追加方針 2026-05-30）。「個人でも会社でも使える」を“ついで”でなく主要メリットに格上げするための専用ビート。料金が「1人でも同額」の根拠を直前で説明する位置でもある。

**信頼補強の位置を確定（copy-strategist 質問への回答・2026-05-30 追記）**:
- 「信頼/セキュリティ」と「AI進化」は **比較表の後・料金の直前** に固定配置する（=信頼補強ブロック）。理由: これらは reassurance（安心づけ）であり、価格を提示する直前＝CTA手前で不安を消す位置が最も効く。堀・主要機能と前半で competing させない。
- **prominence は低**: 独立ヒーロー級にしない。「信頼/セキュリティ」は4アイコンのコンパクトな帯、「AI進化」は1〜2文のミニ band（or 信頼/セキュリティ内に内包）。DESIGN_SYSTEM「1セクション1メッセージ・余白で語る」に従い小さく。
- **省略可の基準**: 縦が長くなりCV率を損なう場合、AI進化は省略可（信頼/セキュリティは残す）。最終の高さ調整は #3 実装者が判断してよいが、**順序は上記で固定**（迷いを残さない）。

**現行順 → 新順の変更**:
- 現行: Hero → 統計バー → 開発者ストーリー → 課題 → 機能4枚 → 信頼/セキュリティ → AI進化 → 比較表 → 料金 → FAQ → CTA → Footer
- 変更: ①Hero を E案へ ②開発者ストーリーを「社会的証明」に拡張（声/Before-After を追加可能な器に）③課題の直後に「お客様連動アプリ」新設（堀を最優先表示）④機能4枚は維持しつつ堀(お客様アプリ)・通話録音を独立化 ⑤通話録音セクション新設 ⑥「信頼/セキュリティ」「AI進化」は比較表の後・料金の直前に小さく統合（独立ヒーロー級にしない・上記で位置確定）。

### 2. 統計バー 【信頼・数値で語る】

### 3. 社会的証明 【層③＋信頼担保】
- 【目標】「この会社・プロダクトは信用できる」と思わせ以降の主張を受け入れやすくする。LP最大の弱点(社会的証明ゼロ)の補強。
- 構成:
  - **開発者ストーリー**（現役の不動産売買営業が自分のために作った＝現場を知っている）。捏造不要の最強カード。現行を維持・主役化。

### 4. 課題提起 【層③→②への橋渡し】
- 【目標】「これは自分の店の話だ」と自分ごと化させ、解決策（堀）を読みたくさせる。
- 切り口（経営者の痛み・3大ムダ）: ①同じ物件を複数媒体に手入力するムダ（※「連携費が高い」より「多重入力のムダ」に主語を寄せる＝00 §5-9）②新人教育の属人化・離職（現行のPainに無い＝追加）③マッチング・追客の属人化と漏れ。
- 現行差分: Pain1 の主語を是正、Pain に「新人教育の属人化」を追加。最後の問いを「追客の漏れ・タイミング」にして堀へ自然接続。

### 5. 堀①: お客様連動アプリ（最優先セクション・新設）【層②・最重要】
- 【目標】**競合にない独自体験を最初に強く印象づけ「これは他にない」と記憶に焼き付ける。** LP全体で最も力を入れるセクション（現行 LP に一切無い＝最大の差別化が未掲載という重大欠落の解消）。
- 説明: 「お客様は自分のスマホで物件を見て、"気になる"を返すだけ。その反応が担当者にリアルタイムで届く。」星評価・メモ・内見希望・掲載終了チェック・AI傾向まとめも。
- ベネフィット: 「お客様の本音が見える」「追客のタイミングを逃さない」「お客様のスマホが、あなたの営業チャネルになる」。
- ビジュアル: お客様スマホ → 担当者ダッシュボードへ反応が"逆流"する双方向フロー図（DESIGN_SYSTEM §8 で SVG 内製指定。線 stroke-surface-200／ノード白カード＋緑アイコン／矢印 primary-400）。言葉以上に堀を伝える LP 最大の武器。
- CTA: 「お客様連動アプリの詳細 →」で /features/customer-app へ回遊。
- 裏取り: CustomerAppPage/CustomerSharePage＋Edge Function customer-app 実在（02 §C）。Siri保存は2026-05-30復旧済（実機保存成功確認）。
- 現行差分: **完全新設**。

### 6. 主要機能 【層①②＋実用価値】
- 【目標】「日々の業務が本当にラクになる／成果につながる」と実感させ堀を支える地力を見せる。
- カード構成（現行の機能4枚を活かしつつ整理）:
  - **コピペ／PDFで物件登録**（主訴求①の実証。レインズ/SUUMO/Excelコピペ＋チラシ・マイソクPDF。入力90%削減）。動画あり。※PDFは現行未掲載＝追加（00 §5-7。ただし"差別化"でなく"同等機能"扱い＝§8/00 §10-3）。
  - **逆引きマッチング**（物件→買いそうな顧客を点数順）。**※売買のみ。賃貸と同時混在表記NG（§8）。** 「双方向」表記より「逆引き」に寄せると分かりやすい。
  - **顧客1人1人に専属AI**（次の一手・ヒアリング・刺さる点・メール/電話トーク）。「楽さ」だけでなく「機会損失を防ぐ＝成果」へ橋渡し。
  - **契約フェーズ・精算・書類**（漏れ防止）。
- 各カードに詳細ページ導線（§4）。
- **業務フロー図（RESEARCH_BRIEF §3/§4 反映・2026-05-30 追記）**: セクション冒頭に「貼る → AIがマッチ → AIが追客」の3ステップ図を置く（いえらぶの5ステップ業務フロー図が全体像を一目で伝えていた＝競合の型を縮約して借用）。SVG内製（DESIGN_SYSTEM §8）。主要機能カードはその3ステップを裏付ける詳細という構成にすると流れが通る。
- 現行差分: 3ステップ業務フロー図を新設、機能カードにPDF登録を追記、逆引きを売買限定で明記、各バッジに成果側のベネフィットを1つ足す。お客様アプリ・通話録音はここから独立済み。

### 8. 比較表 【層②・差別化を一目で】
