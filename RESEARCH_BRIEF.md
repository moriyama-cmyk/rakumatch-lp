**海外勢の盗みどころ（ライブ取得で更新）**:
- **CTAの2系統併置が定番だが、製品タイプで主従が分かれる**。Follow Up Boss（セルフサーブCRM）は「無料トライアル」主＋「デモ」従。Lofty（高機能・高価格）は「デモ予約」主で**無料トライアルを意図的に提供しない**。→ 楽マッチはコピペで即体感できるセルフサーブ＝**Follow Up Boss型（無料トライアル主＋デモ/資料従）が正解**。
- **Follow Up Boss はヒーローで社会的証明を“いきなり”出す**（トップ50中41チーム/10万人/満足度98.2%/目玉事例）。社会的証明の早出しは強力。楽マッチは実績が小さいので同じ強度は不可だが、「現役の不動産営業が開発」＋インフラ信頼で“早めに信頼の根拠”を置く発想は流用できる。
- **「カード登録不要・full access・即サポート」**（Follow Up Boss）は無料トライアルの摩擦を下げる定番文言。楽マッチも「カード登録の要否」を明示すべき。
- 価格は海外も**ヒーローでは非公開・別ページ**が多い（docsの公開値は別ページ由来）。ただし楽マッチは低価格が武器なので、海外慣習に従わず**価格を前面化**する逆張りが効く。
- 「逆方向共有・逆引きマッチング」は Redfin Shared Search / Zillow My Agent / Lofty Buyer Matching として**海外では既存**なので、海外を引き合いに「世界初」と言うのは不可。

## 2. teardown から導く「楽マッチが取り入れるべき/避けるべき」（ライブ取得の知見で更新）

### 取り入れるべき
1. **ヒーロー直下に成果数値3点を置く型**。Facilo（2,500店舗/成約率200%/提案時間80%）も いえらぶ（15,000社/連動率95%/メール作成90%削減）も**ヒーロー周辺に数値を畳みかける**。楽マッチの現行Statsバー（90%/1秒/5秒）の配置は型として正解。ただし**根拠注記が必須**（楽マッチは事例が無いので「※自社利用に基づく試算」）。
2. **社会的証明の「数」を前面に**。Facilo「2,500店舗超＋ロゴ20社」、いえらぶ「15,000社＋実名事例3社＋認証バッジ」。BtoB購買は数と実名で動く。楽マッチは実数が小さい段階→**代替**（属性ラベル付き匿名の声、モニター/ベータの生声、自社Before/After、「現役の不動産営業が開発」、認証/インフラバッジ=Gemini/AWS/Stripe）を同じ位置に置く。社会的証明は「比較表・料金の前」に必ず差す。
3. **CTAを低コミットに分散**。**国内競合は揃って「資料DL/相談」を主CTAにし、サインアップを強制しない**（Facilo「資料を見てみる」、いえらぶ「サービス資料ダウンロード/メールで受け取る」＋電話番号常設）。楽マッチは現状/signup一択。**「資料を見る」「3分の紹介動画」「機能を詳しく」等の低コミット導線を追加**し、いきなり申込まない検討層を拾う（ただし楽マッチはセルフサーブ型＝無料トライアルを主CTAに残しつつ、検討導線を“併置”するのが最適）。
4. **痛み→解決の対句フレーミング**（いえらぶ「ポータルごとに何回も登録するのが面倒 → ワンクリックで出稿」）。楽マッチも「同じ物件を3媒体に手入力 → 貼るだけで終わる」の対句が効く。
5. **機能ごとにスクショ/GIF＋短い効能**を必ず付ける（Facilo・いえらぶとも各機能にスクショ/GIF）。zig-zag機能の各ブロックに「楽さ」だけでなく「成果（機会損失を防ぐ等）」の一文を足す。
6. **5ステップの業務フロー図**（いえらぶ: 登録→出稿→管理→契約→分析）。楽マッチも「コピペ登録→マッチング→提案/追客→案内→契約」の一連を1枚の流れ図にすると、機能群が“一気通貫”に見えて価値が伝わる。
7. **デモ/動画**: 最大USP（コピペ登録）の画面録画を機能1に置く現行方針は正しい（00§6でも固定ポスト化を推奨）。お客様連動アプリ・逆引きも短尺デモ化の価値大。**注**: 国内競合（Facilo/いえらぶ/KASIKA）は意外にも“動画”を使わずGIF/スクショ中心。海外でも Follow Up Boss が前半にデモ動画を置く程度。楽マッチが短尺デモ動画を載せれば体験訴求で一歩出られる（KASIKAは体験を全く見せておらず差別化余地が大きい）。
8. **CTAの主従はセルフサーブ型に合わせる**（海外ライブ取得）。Follow Up Boss は「無料トライアル」主＋「デモ」従、Lofty は高価格ゆえ「デモ予約」主で無料トライアル無し。楽マッチはコピペで即体感＝**無料トライアル主＋資料/デモ従**が正解。「カード登録不要・full access」等の摩擦低減文言を添える。
9. **社会的証明の早出し**（Follow Up Boss はヒーローで「トップ50中41チーム/10万人/満足度98.2%」をいきなり出す）。楽マッチは実績規模が小さいので同強度は不可だが、「現役の不動産営業が開発」＋インフラ信頼を“早めに”置いて信頼の根拠を前倒しする発想は流用可。

### 避けるべき
1. **価格を隠す型は真似しない**。Facilo・いえらぶ・ノマドは**価格非公開**、KASIKAは公開だが**初期7万＋月6万と高額**。楽マッチの「月3,000円/人・初期0円」公開は最大の差別化なので**むしろ前面化**（隠す競合と逆を行く）。KASIKAの「初期7万＋月6万」は比較表の具体的な対比材料として有効（ライブ確定値）。
2. **オールイン機能総量で殴る型**（いえらぶ＝13〜17モジュール）に乗らない。中小には過剰・重い印象。楽マッチは「導入の軽さ×低単価×独自体験」で土俵をずらす。
3. **「成果（成約・売上）の約束」コピーをそのまま真似しない**。Facilo「成約率200%増」、いえらぶ「売上を伸ばす」は**自社実績/既存顧客基盤に裏打ち**された数値。楽マッチは事例が無い＝同じ成果表現は景表法リスク。「動き・プロセスの再現」＋注記に言い換える。
4. **「業界唯一/世界初」表現**は避ける（逆方向共有・逆引きは海外で既存）。「他にあまり見られない逆方向の仕組み」等の相対表現。
5. **MA特化の見せ方**（KASIKA＝休眠掘り起こし/行動スコアリング中心）に寄せない。楽マッチは「現場の営業行動支援に全振り」を強みに。

## 3. 効くCV要素チェックリスト（不動産SaaS LP・実装可否つき）

page.tsx 改修で「入れる/直す」を判断するためのチェックリスト。各項目に【根拠】と【楽マッチ対応】。
※SaaS LP 一般則（ライブ取得）: 訪問者の50〜70%はファーストビューで離脱→**H1とCTAの強さが最重要**。CTAは「FV内/課題提示の下/デモの下/事例の下/最下部」に最低5箇所。「カード登録不要」「3分で始められる」等の補足で心理ハードルを下げる。セルフサーブで価値体感できる製品は無料トライアル、設定が要る製品はデモ予約が向く（楽マッチ＝コピペで即体感＝**無料トライアルが正解**）。

### A. 社会的証明（BtoB SaaSで購買を最も動かす・最優先）
- [ ] **顧客の声/事例を比較表・料金の前に配置**【根拠: Facilo事例6本・いえらぶ実名3社を中盤に厚く配置／03弱点1で最大欠落】【対応: モニター/ベータ生声・自社Before/After・属性ラベル匿名（「東京・売買仲介/一人社長」）でも可。最低1〜2件をWeek0で確保】

### B. 数値・主張の信頼性（誠実性＝逆に信頼を生む）
- [ ] **全実績数値に根拠注記**（90%/年間250時間/月2〜3件）に「※自社利用に基づく試算」【根拠: 03弱点5・00§5／景表法】【対応: 即対応・低コスト・必須】
- [ ] **「成果保証」表現を「行動・プロセス支援」に言い換え**【根拠: 02§6主張3】

## 4. 推奨セクション順（根拠つき）

   - 根拠: 03§5-3 推しE案／一言ポジショニングを忠実再現。バッジ「不動産売買特化のAI営業CRM」。「成果」を使う場合は注記必須。
2. **Stats bar（根拠注記つき）**
   - 根拠: Facilo・いえらぶともヒーロー周辺に数値を畳みかける。型は正解。注記で誠実性担保。
3. **Pain points（3大ムダに再設計・対句で）**: ①多重入力（「同じ物件を3媒体に手入力」→「貼るだけ」）②新人教育の属人化・高離職（現行欠落→追加）③マッチングの属人化。
